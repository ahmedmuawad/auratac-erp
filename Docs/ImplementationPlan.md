# خطة عمل مشروع "برق الرماي" لتكنولوجيا الصيانة - المرحلة الأولى

هذا المشروع يهدف إلى بناء نظام سحابي متكامل لإدارة عمليات صيانة أسلحة الصيد لشركة "برق الرماي"، مع التركيز على الأداء العالي وتجربة المستخدم السلسة (Mobile-First).

## مراجعة المستخدم المطلوبة
> [!IMPORTANT]
> - سنقوم بإنشاء قاعدة البيانات `barq_alramay` آلياً.
> - النظام سيدعم اللغتين **العربية والإنجليزية** بشكل كامل (Localization).
> - بوابة الرسائل القصيرة (SMS) ستكون **Twilio**، مع استخدام كود تفعيل ثابت `123456` في المرحلة الحالية.

> [!NOTE]
> سيتم استخدام خط 'Tajawal' للعربية و 'Inter' أو 'Roboto' للإنجليزية لضمان أفضل مظهر.

## التغييرات المقترحة

### 1. تأسيس المشروع (Foundation)
* إنشاء قاعدة بيانات MySQL باسم `barq_alramay`.
* تثبيت لارافيل 11 (Laravel 11).
* تثبيت واستعراض Livewire 3 و Tailwind CSS.
* إعداد نظام الترجمة (Localization) واختيار اللغة (ar/en).
* إعداد تكوين RTL/LTR الديناميكي.

#### [NEW] هيكل المشروع الأساسي
- إعداد ملف `.env`.
- إنشاء Layout أساسي يدعم `wire:navigate` مع تبديل الاتجاه (Dir) بناءً على اللغة.

### 2. هندسة قاعدة البيانات (Database Architecture)
إنشاء جداول قاعدة البيانات كما هو موضح في وثيقة المتطلبات:

#### [NEW] Migrations
- **users**: إضافة الأدوار (Reception, Technician, QA, Manager).
- **customers**: بيانات العملاء (الهوية الوطنية، الهاتف، إلخ).
- **items**: بيانات القطع والأسلحة.
- **maintenance_cards**: كروت الصيانة وحالاتها.
- **repair_tasks**: مهام الإصلاح وتتبع الوقت.
- **qa_inspections**: فحوصات الجودة.

### 3. الواجهة الأمامية الأساسية (UI Shell)
- إنشاء الـ Sidebar والـ Header المتجاوبين.
- إعداد لوحة التحكم (Dashboard) المبدئية.
